The ORDER block is a block we can use to rearrange the data in a Table. You can think of the ORDER block as a way to sort a Table based on the values in a particular column.

The Table that results from an ORDER block call has the same number of rows and columns, only the order of the rows in the new Table is different.

The ORDER block does not change the original Table because as a reporter it creates a new copy of the Table. This is the case for any DASIS reporter blocks, and you only have to worry about making a permanent change to a Table when you are using command DASIS blocks.

The ORDER block takes in 4 arguments, with the third argument being optional. These arguments are:

  1. A Table
  2. A column label or index to order the Table by
  3. A modifying reporter function to apply to the column argument before sorting (optional)
  4. A boolean to indicate ascending/descending order (True = Descending, False = Ascending)

A few tips for using these arguments properly:

  • The optional third argument can be used to modify the column values before sorting. If you do not want to use the optional argument, you can simply leave it blank (as the block has it empty by default) and the block will work properly.
  • Instead of dragging a or block into the ascending/descending argument, you can click on the argument in the block and choose True or False that way.
